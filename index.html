⚡ 獨家進階機制 (Advanced Features)

有別於基礎的 PPO 實作，本框架加入了多項穩定訓練與加速收斂的工程技巧：

🔍 動態探索增幅 (Exploration Boost)

為避免模型陷入局部最佳解，系統內建了防停滯機制：當模型連續 200 次更新未打破最高分記錄時，會自動將 Entropy Coefficient（預設 0.00075）放大 3 倍，並持續 300 次更新以強制增加探索範圍。

📦 DataLoader 批次更新

在神經網路更新階段，捨棄了簡單的迴圈切片，改用 PyTorch 原生的 TensorDataset 與 DataLoader。每個 Epoch 自動洗牌（Shuffle）並以 256 為 Mini-batch 進行更新，大幅提升梯度下降的穩定性與 GPU 記憶體使用效率。

📈 Z-Filter 狀態正規化

實作了 RunningStat 類別，能在環境互動過程中動態計算觀測值（Observations）的平均值與變異數。將狀態進行 Zero-mean 與 Unit-variance 的正規化處理，並支援極端值截斷（Clip），確保神經網路輸入的數值穩定。

⚙️ 階層式 YAML 配置管理

所有的超參數、環境設定、網路結構皆透過自定義的 Config 類別進行打包，並能自動序列化儲存為 config.yml。支援在測試階段直接讀取權重對應的環境設定，保證實驗具備 100% 的可重現性。

📐 核心演算法公式

本專案嚴格遵守 PPO-Clip 目標函數的定義，透過截斷新舊策略比率 $r_t(\theta)$ 來限制網路更新幅度：

$$L^{CLIP}(\theta) = \hat{\mathbb{E}}_t \left[ \min\left(r_t(\theta)\hat{A}_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon)\hat{A}_t\right) \right]$$

優勢函數（Advantage Function）採用 GAE，結合時間差分誤差（TD Error）$\delta_t$ 計算，有效平衡偏差與變異數：

$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$ $$\hat{A}_t = \sum_{l=0}^{\infty} (\gamma \lambda)^l \delta_{t+l}$$

為了鼓勵探索，損失函數中亦加入了常態分佈的熵（Entropy Bonus），其計算方式實作如下：

def torch_entropy(std):
    _N = std.shape[-1]
    log_prod = torch.sum(torch.log(std + EPS), dim=-1)
    ent = 0.5 * (np.log((2 * np.pi * np.e) ** _N) + 2.0 * log_prod)
    return ent.mean() if ent.dim() > 0 else ent
